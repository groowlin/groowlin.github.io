# Быстрые визуальные правки

## 16 пунктов согласованного скоупа

### Все правки
1. Не трогать логику правого preview-блока (`fixed/portal`, переключение состояния).
2. В правом блоке изменить только скругление медиа под Figma: `20px` (вместо текущего `24px` у `previewCard`).
3. Переработать левый блок по сетке и отступам строго по фрейму Figma.
4. Внешняя геометрия левой части: `padding-left: 120px`, `padding-top: 120px`, вертикальный `gap: 40px`.
5. Контейнер списка кейсов: ширина `360px`, `padding: 20px 40px`, вертикальный `gap: 40px`.
6. Расстояние между кейсами внутри секции: `gap: 8px`.
7. Отступы заголовков секций: нижний `20px`; между секциями предусмотреть верхний отступ `40px`.
8. Верхняя карточка профиля: `padding: 20px 20px`, внутренний `gap: 20px`, `border-radius: 40px`.
9. Мобильную адаптацию на данном этапе не менять.
10. Зафиксировать и применить новые отступы элементов: `120`, `60`, `40`, `20`, `8`, `6`, `4` px по соответствующим блокам.
11. Добавить/актуализировать токены дизайн-системы для домашнего экрана.
12. Цветовые токены: `#F5F5F5` (фон top-card), `#EEEFF2` (line/default), `#A1A1AA` (secondary text), `#000000` (primary text).
13. Токены скругления (кроме бабла): `20px` (preview media), `40px` (top-card), `44px` (avatar mask).
14. Токены типографики для home-вью: семейство `SF UI Text` с fallback-цепочкой, веса `400/500/700/800`, размеры `16/18`, `line-height: 19px`.

### Правки бабла
1. Motion и 3D tilt сохранить как есть, не упрощать и не удалять.
2. Визуальный «воздух» вокруг текста активного кейса выставить `padding: 20px 20px`.
3. Токен скругления бабла: `24px` (`active case bubble`).

## Имплементация дизайна: верхний блок навигации в профиль

### Назначение блока
Этот блок должен визуально работать как «профильная точка входа»: мгновенно идентифицирует автора (аватар + имя), дает быстрый контекст (короткий сабтайтл) и быстрые каналы перехода (иконки + стрелка).

### Геометрия и композиция
1. Использовать отдельный контейнер карточки поверх левой колонки, до секции кейсов.
2. Фон контейнера: `#F5F5F5`.
3. Скругление контейнера: `40px`.
4. Внутренние отступы контейнера: `20px 20px`.
5. Горизонтальный gap между аватаром и текстовым блоком: `20px`.
6. Ширину контейнера удерживать в рамках ширины левой колонки (`360px`).
7. Стрелку разместить по вертикальному центру карточки, прижать справа (`right: 20px`) без сдвига основного текстового блока.

### Аватар
1. Размер аватара: `64x64`.
2. Скругление маски аватара: `44px`.
3. Аватар не должен растягиваться; использовать `cover`.
4. Сохранить визуальный приоритет лица: центрировать по безопасной зоне, чтобы не резать ключевые детали.

### Текстовая иерархия
1. Имя: размер `18px`, вес `700` (или эквивалентный `Bold` в `SF UI Text`), `line-height: 19px`, цвет `#000000`.
2. Подпись/навигационная строка («Обо мне»): размер `16px`, вес `400`, `line-height: 19px`, цвет `#000000`.
3. Межстрочный вертикальный зазор между именем и подписью: `4px`.
4. Текстовый блок должен быть выровнен по левому краю и визуально центрирован относительно аватара.

### Иконки и микронавигация
1. Размер каждой иконки: `16x16`.
2. Расстояние между иконками: `8px`.
3. Блок иконок встроить в строку подписи (после «Обо мне») с промежутком `6px` между текстом и иконками.
4. Иконки использовать визуально «легкие», без лишних обводок и без изменения их оригинальной формы.
5. Цвета иконок не «выравнивать насильно»: брендовые допускаются, если так в макете.

### Стрелка перехода
1. Размер стрелки: `24x24`.
2. Цвет стрелки: нейтральный темный, контрастный к фону.
3. Стрелка не должна влиять на общий поток текста (позиционировать отдельно справа).
4. При hover/focus у карточки допускается мягкий визуальный отклик, но без изменения базовой геометрии.

### Поведение и интеракция
1. Блок может быть единой кликабельной зоной (переход на профиль/`/about`).
2. Hover-состояние: легкое усиление контраста/тени, без скачка размеров.
3. Focus-state обязателен: видимый outline в рамках текущего дизайн-языка.
4. Не отключать текущие motion-паттерны проекта; встроить блок в существующую анимационную последовательность появления.

### Токенизация для верхнего блока
Рекомендуемые токены (имена можно адаптировать под текущую систему):
1. `--home-top-card-bg: #F5F5F5`
2. `--home-top-card-radius: 40px`
3. `--home-top-card-padding: 20px`
4. `--home-avatar-size: 64px`
5. `--home-avatar-radius: 44px`
6. `--home-name-size: 18px`
7. `--home-subtitle-size: 16px`
8. `--home-line-height-tight: 19px`
9. `--home-icon-size: 16px`
10. `--home-arrow-size: 24px`

## Имплементация дизайна: секция с кейсами и заголовками

### Структура секций
1. Секция состоит из заголовка и списка кейсов.
2. На странице должны быть отдельные группы: «Портфолио», «Globus», «Работа в студии».
3. Между группами использовать явный вертикальный ритм, а не случайные маргины у отдельных элементов.

### Контейнер и сетка
1. Общий контейнер секций: ширина `360px`.
2. Внутренние отступы контейнера: `20px 40px`.
3. Вертикальный gap между крупными блоками внутри контейнера: `40px`.
4. Расстояние между кейсами внутри одной группы: `8px`.
5. Глобальный отступ между левой и правой колонкой оставить `60px`.

### Заголовки секций
1. Размер заголовка: `18px`.
2. Вес заголовка: `800` (или `Heavy` эквивалент).
3. `line-height: 19px`.
4. Цвет: `#000000`.
5. Нижний отступ у заголовка до первого кейса: `20px`.
6. Для новых секций выше заголовка закладывать `40px` отступ от предыдущего контента.

### Карточка кейса (обычное состояние)
1. Карточка в потоке списка: вертикальная композиция «название + мета».
2. Название кейса: `18px`, вес `500`, цвет `#000000`, `line-height: 19px`.
3. Мета-строка (год, категория, теги): `16px`, вес `400`, цвет `#A1A1AA`, `line-height: 19px`.
4. Разделители в мета-строке: компактные точки с визуальным интервалом `6px`.
5. Подчеркивание/line под заголовком кейса: `2px`, цвет `#EEEFF2`, скругление дорожки большое (pill-like).

### Карточка кейса (активное/hover состояние)
1. Для активного кейса сохранить «плавающий бабл» как в текущей реализации.
2. Скругление бабла: `24px`.
3. Внутренний воздух вокруг текста: `padding: 20px 20px`.
4. Визуальное состояние бабла: мягкая светлая подложка и тонкая тень без агрессивного контраста.
5. Сохранять motion/3D tilt, но в пределах текущей геометрии карточки (без изменения размеров контейнера списка).

### Ритм вертикальных отступов (ключевые числа)
1. `120px` — стартовые отступы левого блока сверху и слева.
2. `60px` — межколоночный интервал.
3. `40px` — крупный шаг между смысловыми группами.
4. `20px` — внутренние отступы контейнеров/бабла и отступ после заголовков.
5. `8px` — плотный шаг между карточками внутри группы.
6. `6px` — интервал между элементами в мета-строке.
7. `4px` — микроинтервал между строками в текстовых блоках.

### Типографика секции кейсов
1. Основной стек: `SF UI Text`, fallback через системный sans.
2. Названия секций: `18/19`, `800`.
3. Названия кейсов: `18/19`, `500`.
4. Мета-текст: `16/19`, `400`.
5. Не смешивать разные line-height в одном уровне иерархии.

### Что оставить без изменений на этом этапе
1. Не менять мобильный слой и брейкпоинты.
2. Не менять логику переключения preview-контента.
3. Не выносить правый блок из текущей архитектуры `fixed/portal`.
4. Не упрощать и не убирать motion/tilt.
## iOS-like corner smoothing rules

1. Для всех non-pill поверхностей home использовать shape layer поверх текущих radius-токенов.
2. `radius` сохраняет текущую пропорцию элемента, `shape mask` задаёт iOS-like характер угла.
3. Preview media использует `--radius-home-preview` + `--shape-surface-preview-mask`.
4. Top card использует `--radius-home-top-card` + `--shape-surface-top-card-mask`.
5. Avatar mask использует `--radius-home-avatar` + `--shape-surface-avatar-mask`.
6. Active case bubble использует `--home-bubble-radius` + `--shape-surface-bubble-mask`.
7. Pill/capsule элементы (`--radius-pill`, точки-разделители, pill-like дорожки) не переводятся в smoothing system.
8. Все визуальные слои одной поверхности должны использовать одну и ту же shape-геометрию: container, media clipping, glass/highlight overlays.
